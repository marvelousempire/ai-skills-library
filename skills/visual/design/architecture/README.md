# visual / design / architecture

**What belongs here:** Design systems and visual structure decisions — how a product's visual components are organized, how tokens flow, how consistency is maintained at scale. The structural layer of design: what rules govern what everything looks like.

Different from `engineering/architecture/` which is about system and code structure. This is about **visual and experiential structure** — the decisions that make a design system coherent.

Currently houses: `emil-design-eng` (design engineering philosophy — how design decisions are made structurally).
