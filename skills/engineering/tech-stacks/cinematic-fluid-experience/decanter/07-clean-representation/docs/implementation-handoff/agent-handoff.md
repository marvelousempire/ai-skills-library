## Core Point

Future developers or agents must preserve immersion and test the hard parts on the right device or runtime.

## Handoff Constraints

1. Start with the design toggle, ReadyPlay primitives, and hybrid-preview concept.
2. Build the RealityView fluid scene using the native scene architecture pattern.
3. Test on real devices for physics, GPU performance, haptics, sensors, and audio.
4. Preserve the full-screen immersive quality bar; do not collapse the product into static 2D panels.

## For Slice 1 Agents

For the web prototype, translate the same constraints into R3F terms: full viewport scene, minimal HUD, explicit tokens, interaction proof, performance readout, and reduced-motion behavior.
