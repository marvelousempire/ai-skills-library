# Cross-Platform Cinematic Design Doctrine

The iOS immersive sports SOP teaches a larger design rule:

> make digital products feel alive, spatial, responsive, and cinematic.

That principle applies to websites too.

Even when a website does not use Apple APIs like RealityKit, RealityView, HealthKit, or Metal, the same intent should be translated into React, Next.js, Tailwind, Motion, Framer Motion, Three.js, and premium component systems.

---

# iOS-To-Web Translation Map

| iOS / RealityKit Idea | Web / React Equivalent | Website Intent |
|---|---|---|
| Full-screen RealityView | full-screen immersive hero section | open with an experience, not a plain page |
| ViewAttachment HUD | floating React panels / cards | keep interface overlays clean and intentional |
| ParticleEmitterComponent | canvas / WebGL / CSS particles | make events feel alive without clutter |
| Metal bloom / vignette / grain | CSS filters, WebGL post-processing, gradient overlays | create cinematic depth and mood |
| PhysicsBodyComponent | spring-based motion / drag physics | make interactions feel responsive and physical |
| ManipulationComponent | draggable / interactive cards, 3D objects, or orbs | let users inspect stats and features interactively |
| Spatial audio | optional ambient web audio | support demos and guided experiences where appropriate |
| Core Haptics | microinteractions and visual feedback | make clicks, hover, and state changes feel satisfying |
| Cinematic camera dolly | scroll-linked motion / parallax / 3D transforms | create guided storytelling |
| Health/WHOOP data-driven effects | dynamic personalization / live metrics | let data influence the visual environment |

# Website Hero Intent

For You-Sir Juan websites, marketing pages, and dashboards:

1. Start with the best available hero-section inspiration from `https://21st.dev/community/components/s/hero`.
2. Shape the hero around the actual product promise, not generic SaaS copy.
3. Use Motion / Framer Motion only where animation explains hierarchy, state, depth, or momentum.
4. Avoid random animation that does not support the story.
5. Build the first screen to feel like a premium product entrance.
6. Treat the hero as the web equivalent of a RealityView opening scene.

---

# Motion Intent

Use Motion / Framer Motion to create:

- reveal sequences
- staggered feature cards
- smooth hero transitions
- scroll-based storytelling
- hover and press feedback
- modal and drawer transitions
- dashboard state transitions
- animated metrics
- product-tour movement

Do not use motion only because it looks cool.

Motion should clarify:

- where the user is
- what changed
- what matters
- what is interactive
- what the product feels like

When building a website from this ecosystem, agents must ask:

