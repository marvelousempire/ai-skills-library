# Workflows

End-to-end recipes for recurring kinds of work. Each workflow ties together standards + checklists + templates + scripts into one repeatable sequence.

| Workflow | Use when |
|---|---|
| [`build-new-skill.md`](build-new-skill.md) | Adding a new skill, start to ship |
| [`audit-and-elevate.md`](audit-and-elevate.md) | After any meaningful ship |
| [`rip-and-replace-a-tool.md`](rip-and-replace-a-tool.md) | Swapping one tool for another (e.g. Docker Desktop → Colima) |
| [`multi-surface-build.md`](multi-surface-build.md) | Building CLI + MCP + Web UI + Docker for the same engine |
| [`repo-as-study-system.md`](repo-as-study-system.md) | Onboarding yourself / a new agent / a teammate |

Workflows reference standards and checklists rather than duplicate them. They're the "in what order do I do the steps" layer.
