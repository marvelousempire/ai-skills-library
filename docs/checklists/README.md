# Checklists

Operator-facing one-page checklists. Use them before, during, or after the corresponding workflow.

| Checklist | Use when |
|---|---|
| [`new-skill.md`](new-skill.md) | Adding a new skill to `skills/<family>/<slug>/` |
| [`ship.md`](ship.md) | Before commit / merge / push of any substantive change |
| [`cross-reference.md`](cross-reference.md) | After adding a skill, rule, family, container, or template |
| [`container-labels.md`](container-labels.md) | When containerizing a new service |
| [`doctor.md`](doctor.md) | When writing a new doctor script |
| [`after-ship-audit.md`](after-ship-audit.md) | After every meaningful ship (gap audit + elevation pass) |

These are derived from [`docs/standards/`](../standards/) and enforced by [`rules/library/`](../../rules/library/) where applicable.
