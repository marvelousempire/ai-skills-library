# Chapter 1 — First Principles

## Section 1.1 — What the AI Skills Library IS

The AI Skills Library is **the operating brain** for all work done under AVERY GOODMAN.

It is not a collection of files. It is not a folder of scripts. It is a **living product catalog** — a system of discrete, shippable units of intelligence called products. Every lesson learned, every pattern discovered, every process refined, every standard set lives here as a product. Every session that ends by growing this library makes the next session smarter than this one was.

The library is simultaneously:
- A **master report system** — every significant arc produces a report
- A **study system** — every pattern becomes a skill you can invoke by name
- A **training system** — context docs and rules shape AI behavior without being asked
- An **improvement system** — every repeated friction becomes a skill; every repeated mistake becomes a rule
- A **filing system** — the correct permanent home for everything that should not live only in a project repo

## Section 1.2 — The Mission

Every product in the library must make every future session better than the session that created it. The library compounds. That is the only goal.

## Section 1.3 — What a "Product" Is

A product is a discrete, shippable, identifiable unit of intelligence. Products are not files. Products are the things the library manufactures and delivers: behaviors, processes, patterns, standards, checklists, templates, and agents.

## Section 1.4 — The Product Types

| Type | ID Prefix | File | Function |
|---|---|---|---|
| Skill | SK- | `skills/.../SKILL.md` | An invocable process |
| Rule | RL- | `rules/library/.../body.md` | An always-on policy |
| Agent | AG- | `agents/.../AGENT.md` | An autonomous actor |
| Template | TM- | `templates/.../` | A starter structure |
| Doc | DK- | `docs/...` | Reference material |
| Checklist | CK- | `docs/checklists/...` | Binary verification |
